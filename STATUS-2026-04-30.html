The frame

EQ is the layer between systems trade subbies are forced to use that don't talk to each other. Not a SimPRO replacement. Not a Xero competitor. The thing in the middle that means apprentices don't do the same induction four times a week and bookkeepers don't retype timesheets at 8pm on a Friday.

Phase 1 is plumbing. The canonical schemas, the validation engine, the AI mapping layer, the bidirectional EQ Format surface. The plumbing serves specific named moments of pain — when it stops doing that, we've drifted.

Pricing read confirmed this session: same product, deeper integrations as the crew grows. A 5-person crew uses Cards for inductions and Format to clean up one supplier's CSV per quarter. A 200-person crew uses both plus the canonical spine, plus client-portal exports, plus accounting write-back. No tiered SKUs, no feature gates — EQ does more for you when you connect more systems. Phase C — power-on session (5 → 8)

5 Real AST-depth fix in cross-field-eval

The historical MAX_DEPTH = 32 counted parser recursion, not AST node depth. A chain like a==1 AND b==1 AND ... AND h==1 has parser recursion of about 5 but AST depth of 9 — so deeply-nested-but-flat chains slipped through. Replaced with a post-parse astDepth() walker and MAX_AST_DEPTH = 8. New test asserts the boundary (9-deep throws, 8-deep compiles).

174 tests passed · typecheck clean

6 noUncheckedIndexedAccess: true in @eq/validation

33 error sites fixed across coerce-date (regex match groups), coerce-number (accounting-negative match), cross-field-eval (tokeniser src[i] with bounded loops, parser tokens[pos] with eof sentinel, getField path traversal), signature-hash (non-empty-checked sorted entries), validate (rows[rowIdx] inside bounded loop, FK ref .split('.')[0]). Non-null assertions only where the invariant is explicit in code. Zero as casts. The validation package's type guarantees now match every other package in the workspace.

174 tests passed · typecheck clean

7 Reshape-out demo — canonical register → next month's PPM SOW

New folder demos/reshape-out-ppm-sow/ with synthetic 20-asset register × 4 generic sites × annual PPM schedule × May 2026 visit-day allocation. derive.mjs (pure Node ESM, no deps) computes which assets need which tasks on which day per visit, emits a sow-asset-schedule.csv with ☐/— tickboxes per applicable task, plus a sow-summary-<site>.csv form per visit.

Output shape matches real PPM SOW spreadsheets — same columns, same tickbox pattern, same per-site summary template. Generic placeholder names only (Site-Alpha / Bravo / Charlie / Delta). README ties it to the conduit thesis: same product for a 5-person crew with one client and a 200-person crew with thirty — second crew just has more rows.

Pure Node · zero deps · output byte-clean

8 Local "save mapping as template" in @eq/format-ui

Two new endpoints: POST /api/templates/find and POST /api/templates/save. Both compute the signature hash server-side via @eq/validation's computeSignatureHash and persist to .templates/<hash>.json (gitignored).

UI flow: every CSV ingest first checks for a template match. If hit, the saved mapping applies directly and a green banner shows "Template matched — no AI call this time." If miss, the heuristic mapper runs as before. "Save as template" button persists the current mapping for next time.

First end-to-end demo of the signature-hash cache hypothesis from EQ-INTAKE-ARCHITECTURE.md. The local-file shape is compatible with the future eq_intake_find_template_by_signature Supabase RPC — when the canonical project lands, the swap is two URL changes.
