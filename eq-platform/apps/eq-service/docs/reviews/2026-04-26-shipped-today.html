Sprint 3.2 · Build
Defects — auto-population from failed test items
Shipped

Migrations 0061 + 0062 add four database triggers:

One on maintenance_check_items, one each on acb_test_readings, nsx_test_readings, and test_record_readings. When a row flips to fail, a defect is auto-created. When it un-fails, the defect auto-resolves with an audit note. Severity by label keyword (Visual=low, Functional=medium, Electrical=high) via the fn_severity_from_reading_label helper. Source columns + back-pointers prevent duplicates.
