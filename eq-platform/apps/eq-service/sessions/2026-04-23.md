## Phase 2 — Work Order Details Template (COMPLETED ✓)

**Goal:** Create new `work-order-details.ts` generator producing per-asset Maximo-parity pages.

### New File: lib/reports/work-order-details.ts

Implements `generateWorkOrderDetailsReport(input: WorkOrderDetailsInput): Promise<Buffer>`

#### Key Features

- **One page per asset** (multi-asset checks produce multi-page reports with page breaks)
- **Masthead** (customer + tenant logos, "Work Order Details" caption)
- **Title block**: "Work Order Details" (large, bold, primary color) + WO# as subhead
- **Asset identification**: Name, location, job plan type ("Low Voltage Air Circuit Breaker (ACB)")
- **2-column info grid** (6 rows):
  - Status, Work Type | Priority, Job Plan
  - Crew ID, Failure Code
  - Target Start, Target Finish | Actual Start, Actual Finish
  - IR Scan, Classification
  - _All fields default to `—` when null (pre-Phase-3 behavior)_
- **Problem/Cause/Remedy section**: Narrative fields for failure context
- **Tasks table**: Task ID | Description | Pass | Fail | N/A | Comments (supports `maximo_task_id` from Phase 3)
- **Technician capture block**: Blank form lines for Name, Date, Hours, Signature, Comments
- **Defects raised section**: Lists linked defects with severity, status, WO# linkage
- **Footer**: `{companyName} — Work Order Details — rev 3.1` (left) + `Page X of Y` (right)
